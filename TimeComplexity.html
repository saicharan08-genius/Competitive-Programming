Time Complexity

What is Time Complexity?

Time Complexity is the number of operations an algorithm runs.

Why do we need Time Complexity?

In many contests like the USACO contest want you to compile your C++ code in 2 seconds and Python/Java Code in 4 seconds whereas in CSES contest or Codeforces Contest, whatever be your language, your code should compile in 1 second. Thus in competitive programming Time Complexity is needed to Optimize the compiling time in tough contests like USACO, CSES, Codeforces. If your code does not compile in fiven time limit, you get a Time Limit Exceeded Error

What is Best Case Time Complexity?

Best Case Time Complexity is the fastest way a program runs in which optimal inputs are chosen. Let us take an example of the best case Time Complexity code for Sorting Algorithm. It would be for a data structure which is already stored

What is worst case Time Complexity?

The slowest Time needed to complete with pessimal inputs chosen. Worst Case Time Complexity is Always chosen so that programmers know the largest time the code could take doesn't exceed the time limit for compiling.
